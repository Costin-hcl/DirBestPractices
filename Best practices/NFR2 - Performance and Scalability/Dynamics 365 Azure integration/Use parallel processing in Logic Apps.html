Use parallel processing in Logic Apps

Be aware that the "foreach" loops in Logic Apps run by default in parallel. This can improve performance when operations executed on each member of the iterated collection can run independently.

It can also create tricky race conditions if used incorrectly, for instance setting a variable in parallel, since the order of setting the variable is unpredictable.

The big gotcha when using parallel loops is that nested loops will run sequentially, which may cause unnecessary delays. To run operations in parallel for items in a nested loop, create and call a child logic app.
