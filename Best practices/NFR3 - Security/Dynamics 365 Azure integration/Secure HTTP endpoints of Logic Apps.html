Securing HTTP endpoints in Logic Apps

Logic Apps and Azure Functions with HTTP triggers can become a security concern since the URL is accessible from anywhere (although, in fairness, the URL does include a shared access signature). Moreover, if data from a secured system is returned in the HTTP response, this offers a way to expose sensitive information by by-passing authentication. To mitigate this risk, we have a few options:

  • Allow access only from other Logic Apps - if a Logic App is designed as a worker/child to be invoked from others, this setting will increase security while not imposing further restrictions
  • Require authenticated calls - OAuth supported
  • Restrict IP range - see how to do it through the Azure Portal or ARM templates.
