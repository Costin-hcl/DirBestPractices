Using prefixes and extensions

Use a client-specific three-letter prefix when naming any new AOT objects (i.e. Tables, Classes, Data entities, etc.). The prefix needs to be agreed at the start of the project.

As an example, if the agreed project prefix is ABC and we're creating a new table labeled Weather history, the AOT name should be: ABCWeatherHistory. 


When extending a table, you need to change the default extension name, ".extension", to something unique. That is because a table can be extended from multiple modules, so each developer much choose a different extension name.

A good choice is to use the same prefix used on the client's project and also the functional area. As an example, when extending CustTable from the functional part about weather of project ABC, the extension name should be: CustTable.ABCWeather.
