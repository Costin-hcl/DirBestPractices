Custom forms and menus should be similar to standard ones

New forms developed in D365FO should have a similar UI to existing ones. This is enforced through Form patterns, which provide a standard structure and a minimal list of controls.

The Form Patterns report, which can be extracted from a DEV environment using Visual Studio, can help identify the pattern used by standard forms with similar functionality. See the Microsoft documentation for a list of form patterns.

Similarly, newly created menu items should be placed in the appropriate section: Workspaces, Inquiries and reports, Periodic tasks, Setup, etc. The development environment doesn't enforce such a structure, so the developers and functional designers are responsible to choosing a logical menu structure.
