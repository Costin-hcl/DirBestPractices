Reusability
Employing reusable components

When gaps are identified on a project, it's best practice to check if the needed functionality has already been implemented. This could be in:

  • Microsoft packages
  • ISV packages
  • Open-source projects

The licensing and costs of reuse as well as the support and maintenance for any 3rd-party software need to be clearly understood before a decision is made to use them.

In addition to external components, companies may use internal catalogs of products to enable code reuse.

The key behaviors to keep in mind are:

  • Before starting a development, check if the function has been implemented before
  • After developing something that looks reusable, publish it either externally (e.g. GitHub) or internally
  • Evangelize the internal products so that various departments in the company learn about the existing products
  • In the sales process, ensure reusable products are taken into consideration if applicable
